Is this a fake Realtek Wifi dongle?

Published on: 2024-12-23T17:33:18Z

Summary: This doesnt make any sense to me and seems that it shouldnt be possible. I bought some, supposedly RTL8821CU USB wifi adaptors. If I put these in a Proxmox N100 machine and run lsusb then they report as Bus 001 Device 018: ID 0bda:1a2b Realtek Semiconductor Corp. RTL8188GU 802.11n WLAN Adapter (Driver CDROM Mode) lsusb -v -d 0bda:1a2b includes idVendor 0x0bda Realtek Semiconductor Corp. idProduct 0x1a2b RTL8188GU 802.11n WLAN Adapter (Driver CDROM Mode) bcdDevice 2.00 iManufacturer 1 Realtek iProduct 2 DISK iSerial 0 But, on a Raspberry Pi 4, lsusb reports them as Bus 001 Device 008: ID 0bda:c820 Realtek Semiconductor Corp. 802.11ac NIC lsusb -v -d 0bda:c820 includes idVendor 0x0bda Realtek Semiconductor Corp. idProduct 0xc820 bcdDevice 2.00 iManufacturer 1 Realtek iProduct 2 802.11ac NIC iSerial 3 123456 Looking up the ID "0bda:c820" this does appear to match the expected RTL8821CU chipset (with a suspicious serial number!). Is this an 8188GU (or something else) pretending to be a 8821CU? How can the reported device IDs change like this? N100 Proxmox: Linux prox1 6.8.12-4-pve #1 SMP PREEMPT_DYNAMIC PMX 6.8.12-4 (2024-11-06T15:04Z) x86_64 GNU/Linux Pi 4: Linux DesktopPi 6.6.51+rpt-rpi-v8 #1 SMP PREEMPT Debian 1:6.6.51-1+rpt2 (2024-10-01) aarch64 GNU/Linux Ive searched for fake Realtek wifi adapters and various other searches but couldnt find anything that seemed to apply. Any ideas? Thank you
