Extract signer information from portable executable (PE)

Published on: 2024-12-25T16:40:19Z

Summary: Im trying to analyse a piece of malware in form of an .exe I dont really want to put this file in a windows system for obvious reasons. To gain more insight Id like to look at the signer information. Using binwalk I confirmed theres a PKCS7 embedded: $ binwalk –signature -o 78047102 malware.exe DECIMAL HEXADECIMAL DESCRIPTION ——————————————————————————– 78047102 0x4A6E77E Object signature in DER format (PKCS#7), header length: 4, sequence length: 10149 78047188 0x4A6E7D4 DER SHA256 hash 78047243 0x4A6E80B Certificate in DER format (x509 v3), header length: 4, sequence length: 1524 … more certificates inside the PKCS7 Now I try to get binwalk to give me the PKCS7 (or anything, really), but it fails: $ binwalk -e -o 78047102 malware.exe DECIMAL HEXADECIMAL DESCRIPTION ——————————————————————————– WARNING: One or more files failed to extract: either no utility was found or its unimplemented Is there another way to get binwalk to do what I want? Perhaps ghidra hides this functionality somewhere?
