Debian Bookworm always sets COLUMNS to be a little less than the actual terminal width

Published on: 2024-12-25T05:42:32Z

Summary: On Debian Bookworm, the $COLUMNS bash variable is always a little smaller than the actual terminal width. This leaves a little space on the right side whenever I run a program that runs through less, such as man, git help commit, etc. Heres a screenshot of the output of man bash, note the margin on the right side: However, $COLUMNS on my Fedora is set to the full width of the terminal emulator. How can I make the $COLUMNS variable always equal to the full width of the terminal emulator? Since there are so many programs that would go through the pager, I prefer not using a solution that sets COLUMNS= before a particular program launches. The solution should also be able to change $COLUMNS when the terminal window resizes, like what bash does by default.
